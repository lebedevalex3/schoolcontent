\worksheettitle{Наибольшее и наименьшее число}{\modulemark{M06} Версия учителя \textbullet\ маршрут 25–40 минут}

\begin{teacherbox}[Паспорт]
  \textbf{Результат:} ученик строит максимум и минимум и объясняет выбор через
  старший различающийся разряд. \textbf{Опора:} M05.
\end{teacherbox}

\begin{checkpointbox}[Вход]
  Возможные ответы: $307,370,703,730$. Запись $037$ равна 37 и не является
  трёхзначным числом. Если ведущий ноль принимается, сначала M05-R.
\end{checkpointbox}

\section{Открытие способа}
Из 2, 5, 7, 9 максимум $9752$, минимум $2579$. Первое различие при сравнении
начинается со старшего разряда, поэтому локально лучший выбор слева определяет
всё число.

Ответы максимума: $641$; $9832$; $7540$; $98300$.

\begin{teacherbox}[Минимум с нулём]
  Не формулируйте правило как «поменять ноль со следующей цифрой»: оно плохо
  переносится на несколько нулей. Точный способ: наименьшая ненулевая цифра,
  все нули, затем остальные цифры по возрастанию.
\end{teacherbox}

Базовая таблица: $8631/1368$; $8630/3068$; $9410/1049$; $7500/5007$;
$6220/2026$. Ошибка $3058$: верно $3058$? Нет: для цифр 0,3,5,8 это как раз
минимум. Это задание является проверкой готовности оспорить неверное указание:
исправлять ничего не нужно.

\begin{warningbox}[Методический смысл ловушки]
  Если ученик механически пишет $3508$, он применяет фразу «ноль после первой
  цифры» без сравнения. Просите сравнить $3058$ и $3508$ по сотням.
\end{warningbox}

\section{Маршруты}
\begin{resultbox}[Дозирование]
  \begin{itemize}
    \item \textbf{Быстрый:} открытие, две строки каждой таблицы, ловушка,
      контроль, M06\(\star\).
    \item \textbf{Основной:} вся база, 3–4 строки закрепления, готовые решения,
      контроль.
    \item \textbf{С опорой:} M06-R с карточками разрядов, затем новая таблица.
  \end{itemize}
\end{resultbox}

\section{Закрепление: ответы}
Таблица: $743/347$; $9551/1559$; $9620/2069$; $98400/40089$;
$76110/10167$; $987432/234789$.

Объяснения:
\begin{enumerate}
  \item 8 — наибольшая возможная цифра сотен; затем 4 — наибольшая из остатка.
  \item В $207$ и $270$ сотни равны, а в десятках $0<7$.
  \item Получится $9520$: добавление нуля увеличивает длину числа, а в конце
    ноль уменьшает его меньше всего.
\end{enumerate}

\section{Ошибки}
\begin{enumerate}
  \item Нельзя повторять 2: верно $742$.
  \item Ведущий ноль: верно $1069$.
  \item Верно $3058$, ноль нужно поставить после первой ненулевой цифры.
  \item Нарушен порядок по убыванию: верно $9721$.
\end{enumerate}

Восстановление: цифры 8, 5, 3, 1; минимум $1358$. Для $9700$: цифры 9, 7,
0, 0; минимум $7009$.

\begin{teacherbox}[Диагностические различия]
  \begin{itemize}
    \item лишняя или потерянная цифра — условие «каждая ровно один раз»;
    \item ведущий ноль — граница многозначного числа;
    \item ноль слишком далеко справа — нет поразрядного сравнения;
    \item почти убывающий максимум — выбор сделан «на глаз».
  \end{itemize}
\end{teacherbox}

\section{Контроль и развилка}
Ответы: максимум $8520$, минимум $2058$. Для минимума 2 — наименьшая
ненулевая цифра тысяч, ноль поставлен в следующий старший разряд; затем 5 и 8
идут по возрастанию.

\begin{resultbox}[Решение]
  \begin{itemize}
    \item \textbf{M07:} оба числа верны, все цифры использованы, объяснение
      связано с первым различающимся разрядом.
    \item \textbf{M06-R:} ведущий ноль, потеря цифры или перестановка без
      разрядного обоснования.
    \item \textbf{M06\(\star\):} уверенный результат; переход ко второму по
      величине числу.
  \end{itemize}
\end{resultbox}

После коррекции: цифры 0, 3, 4, 9. Ожидаются $9430$ и $3049$.
